# Bella Baxter — Python SDK Samples

Samples showing how to integrate Bella Baxter secrets into Python applications.

| Sample | Approach | Best for |
|--------|----------|---------|
| [01-dotenv-file](./01-dotenv-file/) | `bella secrets get -o .env` + `python-dotenv` | Simplest setup, CI/CD scripts |
| [02-process-inject](./02-process-inject/) | `bella run -- python app.py` | Any command, no SDK code |
| [03-flask](./03-flask/) | SDK in app factory (`create_app`) | Flask web apps |
| [04-django](./04-django/) | `AppConfig.ready()` hook | Django web apps |
| [05-fastapi](./05-fastapi/) | `lifespan` async context manager | FastAPI / ASGI apps |

---

## Which approach to choose?

```
Do you want zero Python SDK code?
├── YES → 01 (.env file) or 02 (process inject)
│
└── NO → Do you need live secret values in the app process?
    ├── Flask  → 03-flask  (sync SDK, app factory)
    ├── Django → 04-django (sync SDK, AppConfig.ready)
    └── FastAPI / async → 05-fastapi (async SDK, lifespan)
```

## Typed Secret Code Generation

`bella secrets generate python` fetches the secrets manifest (key names + type hints, no values) from the Bella API and generates a typed `AppSecrets` class. Each property reads from `os.environ` at runtime — no secret values are ever embedded in the generated file.

```bash
bella secrets generate python -o app_secrets.py
```

**Generated `app_secrets.py`:**

```python
# Auto-generated by bella secrets generate python — do not edit manually.
import os

class AppSecrets:
    @property
    def database_url(self) -> str:
        v = os.environ.get("DATABASE_URL")
        if v is None:
            raise RuntimeError("SECRET 'DATABASE_URL' is not set.")
        return v

    @property
    def port(self) -> int:
        v = os.environ.get("PORT")
        if v is None:
            raise RuntimeError("SECRET 'PORT' is not set.")
        return int(v)

    @property
    def enable_feature_x(self) -> bool:
        v = os.environ.get("ENABLE_FEATURE_X")
        if v is None:
            raise RuntimeError("SECRET 'ENABLE_FEATURE_X' is not set.")
        return v.lower() in ("1", "true", "yes")
```

### Usage alongside existing SDK patterns

After loading secrets into the environment (via any of the samples above), use the generated class for type-safe access:

```python
# 02-process-inject or 03-flask already loads env vars via bella run / SDK
from app_secrets import AppSecrets

secrets = AppSecrets()
db_url = secrets.database_url   # str — raises if missing
port   = secrets.port           # int — parsed automatically
```

Because each property reads `os.environ` on every access, secrets are always up to date.

### Options

| Option | Default | Description |
|--------|---------|-------------|
| `-p, --project <slug>` | `.bella` context | Project slug |
| `-e, --environment <slug>` | `.bella` context | Environment slug |
| `--provider <slug>` | `default` | Provider slug |
| `-o, --output <path>` | `app_secrets.py` | Output file path |
| `--class-name <name>` | `AppSecrets` | Class name |
| `--dry-run` | — | Print to stdout without writing |
